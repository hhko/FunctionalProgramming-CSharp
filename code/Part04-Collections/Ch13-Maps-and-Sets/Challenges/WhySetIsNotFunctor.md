# 챌린지 ① — Set 은 왜 (무제약) Functor 가 아닌가

`Map<K,V>` 에는 Functor 가 깔끔히 붙었는데 `Set<A>` 에는 왜 안 붙는가? 두 근거를 직접 설명하라.

## 근거 1 — 시그니처 제약 충돌

`Functor<F>` 의 시그니처는

```csharp
static abstract K<F, B> Map<A, B>(Func<A, B> f, K<F, A> fa);
```

로 `A` / `B` 에 *아무 제약이 없다*. 그런데 `Set<B>` 를 만들려면 결과 원소들을 다시 *중복 제거* 해야 하고, 그러려면 `B` 에 `IEquatable<B>` 또는 `EqualityComparer<B>` 가 필요하다. 무제약 `Map<A,B>` 시그니처 안에서는 이 제약을 걸 수 없다 — `K<F, A>` 인코딩의 구조적 한계.

## 근거 2 — 모양 보존 (Functor 법칙) 위반

Functor 의 `Map` 은 *원소 개수를 바꾸지 않아야* 한다 (모양 보존). 하지만 집합에서

```
{1, -1, 2}.Map(x => x * x) = {1, 4}      // 원소 3개 → 2개
```

처럼 서로 다른 원소가 같은 값으로 가면 *개수가 줄어든다*. 항등·합성 법칙을 형식적으로는 통과해도 "구조 보존" 직관을 깬다.

## 그래서 — Set 은 Foldable

읽기만 하는 `Foldable` 은 위 두 문제가 없다 (원소를 새로 만들지 않으므로 제약도, 개수 변화도 없음). 그래서 `MySet` 은 `Foldable<SetF>` 만 부착했다.

**연습** — `MyMap<Key, V>` 의 `Map` 은 왜 이 두 문제를 피하는가? (힌트: 변환되는 것은 *값 V* 뿐이고, *키 Key 는 그대로* 라 개수가 보존된다.)
