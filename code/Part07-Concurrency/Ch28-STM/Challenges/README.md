# Ch28 챌린지

트랜잭션 메모리(STM/Ref)를 다룬다. *필수* 한 개 + *심화* 한 개.

## 필수 챌린지

### ① 계좌 이체의 원자성

`Bank.cs` — 출금과 입금을 한 STM 트랜잭션으로 묶는다. 둘은 *함께* 커밋되고(all-or-nothing), 중간 상태(한쪽에서 빠졌는데 아직 안 들어온)는 다른 트랜잭션에 절대 보이지 않는다. 동시 이체에도 *총액이 보존* 된다.

**노리는 능력** — 여러 `Ref` 를 한 트랜잭션으로 묶는 STM 의 핵심. `STM<A>` 가 `Monad` 라 LINQ 로 합성된다.

## 심화 챌린지 (선택)

### ② 낙관적 동시성 + 재시도

`STMOps.Atomically` — 본문을 실행해 읽기/쓰기 로그를 쌓고, 글로벌 락에서 읽은 버전을 검증해 충돌이면 *전체 재시도* 한다. 왜 이게 락으로 두 계좌를 직접 잠그는 것보다 *교착 없이* 안전한지 설명하라.

**노리는 능력** — 27장 Atom 의 CAS 재시도가 *여러 참조의 트랜잭션* 으로 확장된 것이 STM 임을 본다.
