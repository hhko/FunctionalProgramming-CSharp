# Ch27 챌린지

원자적 참조(Atom / CAS)를 다룬다. *필수* 한 개 + *심화* 한 개.

## 필수 챌린지

### ① 잃어버린 갱신 없는 동시 증가

`Program.cs` — 여러 스레드가 `Atom.Swap(c => c with { N = c.N + 1 })` 로 동시에 증가시켜도 최종값이 정확히 `스레드수 × 횟수` 다. 같은 일을 락 없는 `int++` 로 하면 갱신이 *유실* 된다 (비교).

**노리는 능력** — CAS 루프가 락 없이 안전함을, 그리고 갱신이 *순수 함수* 라 충돌 시 재적용해도 안전함을 본다.

## 심화 챌린지 (선택)

### ② 순수 갱신의 중요성

갱신 함수에 부수 작용(예: 외부 카운터 증가)을 넣으면 CAS 재시도 때 *중복 실행* 되어 깨진다. 왜 Swap 의 함수가 순수해야 하는지 설명하라.

**노리는 능력** — "상태 변화를 순수 함수로" 라는 7부의 발상이 동시성 안전성의 전제임을 이해한다.
